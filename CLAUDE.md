# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm start          # dev server at localhost:3000 (hot reload)
npm run build      # production build → build/
npm run serve      # serve the production build locally
npm run clear      # clear Docusaurus cache (fixes stale build issues)
```

No test suite — this is a pure documentation/presentation site.

## Environment Variables

| Variable | Default | Purpose |
|---|---|---|
| `DOCUSAURUS_CONF_URL` | `http://localhost:80` | Sets the site URL in `docusaurus.config.js` |
| `DOCKER_IMAGE_TAG` | `latest` | Shown in the footer copyright line |

## Architecture

This is a **Docusaurus 2** site that serves as a portfolio/demo showcase for a DevSecOps pipeline project (the pipeline itself lives in a separate repo).

**Content flow:**
- `src/data/project.json` — single source of truth for all homepage content: hero text, badges, pipeline stage timeline cards, and GitHub link. The homepage (`src/pages/index.js`) reads this JSON and renders it; to update content, edit the JSON, not the JSX.
- `docs/pipeline.md` — the full pipeline architecture doc, rendered under `/docs/pipeline`. The sidebar is autogenerated from the `docs/` directory.
- `blog/` — security blog posts in MDX format with `authors.yml` for author metadata.

**Homepage layout (`src/pages/index.js`):**
- `HomepageHeader` — hero section with name/title, description paragraphs, security tool badges, GitHub button, and a terminal-card CTA linking to `/docs/pipeline`.
- `PipelineStages` — vertical timeline (via `react-vertical-timeline-component`) driven entirely by `projectData.stages`. Each stage entry uses `type: "build" | "security"` to choose the icon. Description lines starting with `- Label: item1, item2` are rendered as tech-stack pills; plain strings render as prose.

**Styling:**
- `src/css/custom.css` — global Infima overrides plus all custom classes. Light/dark theme uses separate CSS variable sets. Timeline date labels use `.contrastWithBackground` with a media query override for mobile (≤1169px) where the date is inside a light card.
- `src/pages/index.module.css` — scoped styles for the homepage components only.

**Config:** `docusaurus.config.js` reads `DOCUSAURUS_CONF_URL` and `DOCKER_IMAGE_TAG` at build time — these must be set in the environment for Docker/production builds.
